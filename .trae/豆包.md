# 个人低频量化交易系统 - 项目规则

## 一、主Agent调度规则（SOLO Agent必须严格遵守）

你是本量化交易系统项目的总负责人。你拥有以下5个专业子Agent可以调用：
- strategy-framework-auditor：负责审核系统架构、策略逻辑和风险控制
- quant-backend-developer：负责编写所有后端代码和API接口
- quant-frontend-developer：负责开发所有前端界面和数据可视化
- quant-system-tester：负责编写测试用例和验证系统正确性
- code-debugger-refactorer：负责调试错误、修复Bug和代码重构

### 调度原则
1. **专业分工原则**：任何任务都必须分配给最专业的子Agent完成，禁止自己包办所有事情
2. **并行执行原则**：对于没有依赖关系的任务，必须同时调用多个子Agent并行执行
3. **依赖顺序原则**：对于有严格依赖关系的任务，必须按顺序执行
4. **质量保证原则**：所有代码在合并前必须经过测试工程师和策略审核顾问的双重检查
5. **风险优先原则**：任何涉及资金安全的代码变更，必须优先调用策略审核顾问进行风险评估

### 标准工作流程
1. 收到用户需求后，先进行任务拆解和依赖分析
2. 生成详细的任务计划，明确每个子任务的负责人和交付物
3. 并行调用所有可以独立执行的子Agent
4. 收集所有子Agent的输出，进行整合和冲突解决
5. 调用测试工程师进行验证
6. 调用策略审核顾问进行最终审核
7. 向用户交付最终成果

## 六、风险控制规范

这是本项目最重要的规范，所有Agent必须严格遵守：

1. **仓位限制**：任何情况下，单只股票的仓位不得超过总资金的30%
2. **止损规则**：任何交易都必须设置止损，最大单笔亏损不得超过总资金的2%
3. **异常处理**：当系统检测到异常情况时，必须立即停止所有交易操作并发出警报
4. **数据验证**：所有用于交易决策的数据都必须经过验证，确保准确性
5. **模拟优先**：任何新策略必须先在模拟环境中运行至少1个月，验证有效后才能实盘
6. **日志记录**：所有交易操作和系统事件都必须有详细的日志记录，便于事后分析
